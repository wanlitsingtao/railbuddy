💡 使用说明

  • 分类名称：工程系统分类名称，如"通信系统"、"信号系统"等，将用于中标记录的工程分类字段。
  • 关键词：用逗号分隔。抓取到的公告标题或描述中包含任一关键词时，会自动归类到该分类。
  • 启用/禁用：禁用的分类不会参与自动分类匹配，但仍保留配置。
  • 排序：数字越小显示越靠前，分类匹配时按顺序优先匹配。
  • 系统默认分类已在初始化时预置了轨道交通常见子系统分类，您可根据需要增删改查。

添加网站数据源

×
网站：通过 HTML 抓取列表页 + CSS 选择器提取条目，适用于普通网站。
API：通过 JSON 接口直接获取结构化数据，适用于 SPA 动态加载站点（如深圳公共资源交易中心）。
给这个数据源起个容易识别的名字，如"XX市公共资源交易中心"
招标公告列表页面的完整网址。打开目标网站，找到公告列表页，复制浏览器地址栏中的 URL。
示例：青岛公共资源交易中心列表页 https://ggzy.qingdao.gov.cn/Tradeinfo-GGGSList/0-0-0
用 CSS 选择器定位列表中的每一条公告。获取方法：在列表页右键 → 检查元素 → 找到包含单条公告的 HTML 元素 → 右键「Copy → Copy selector」。
常用形式：ul.list li、table tr、div.items div.item
如果公告链接是相对路径（如 /detail/123.html），需要填写网站域名作为前缀，系统会自动拼接成完整 URL。
示例：https://cg.shenzhenmc.com。如果链接已经是完整 URL 则留空。
在"列表选择器"找到的每个元素内部，用来定位标题文字的选择器。通常是 a（链接文字就是标题）。
如果标题在 h3 里：填 h3
在列表项内部定位详情页链接的选择器。通常也是 a，系统会取其 href 属性。
在列表项内部定位发布日期的选择器。如果页面没有显示日期，留空即可。
示例：span.date、td:last-child（表格最后一列）
如果勾选了"抓取详情页内容"，系统会打开每条公告的详情页，用这个选择器提取正文内容。一般不需要开启。
示例：.content、#articleBody
勾选后系统会逐个打开公告详情页抓取正文。会增加请求量和耗时，一般不需要开启。
JSON 接口的完整地址。系统会向该地址发送请求获取列表数据。需要在浏览器中按 F12 → Network 找到该 API。
API 通常按频道 ID 区分不同栏目（如建设工程、政府采购）。在 API 文档或请求包中找到。
最多抓取的页数。增量抓取时只取首页（首次除外）。
部分站点证书过期或自签，需取消勾选。
在 API 返回 JSON 中，定位到列表数组的路径。示例：data.content、result.list、data.items
用 {id_field} 占位符生成详情页完整 URL。示例：https://new.szggzy.com/jygg/details.html?contentId={id}
如 API 直接返回分类名称（如"招标公告"），填此项可在下方做映射；不填则用标题关键词分类。
把 API 返回的分类名映射为统一分类（招标/中标/变更/新闻/招标计划/开通运营）。每行格式：原始名=统一名。不填则用标题关键词分类。
系统只抓取标题中包含这些关键词的公告。用逗号分隔多个关键词，满足任一关键词即匹配。
建议：招标,采购,中标,成交,公告,地铁,轨道,城轨,开通,运营
请求该网站的超时时间。如果网站响应慢可适当增大。
连续两次请求之间的间隔，避免请求过快被网站拦截。建议 ≥ 2 秒。
添加后可先设为"否"，测试成功后再启用。
